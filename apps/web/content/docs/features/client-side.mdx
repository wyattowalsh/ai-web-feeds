# Client-Side Power Features

AI Web Feeds implements powerful client-side features that run entirely in your browser, requiring zero backend infrastructure. All features work offline and respect your privacy.

## 🚀 Key Benefits

- **Zero Infrastructure Costs**: No servers, databases, or hosting fees
- **Instant Performance**: No network latency, all operations local
- **Complete Privacy**: Data never leaves your device
- **Offline-First**: Works on planes, trains, and poor connections
- **Unlimited Scale**: Each user's browser handles their own data

## 📱 Offline Feed Reading

### Overview

Save feeds for offline reading with automatic background sync when you're back online. Perfect for mobile users with intermittent connectivity.

### Features

- **Service Worker Caching**: Automatically caches feed articles for offline access
- **Background Sync**: Syncs new articles when connection is restored
- **Conflict Resolution**: Smart conflict detection and resolution UI
- **Storage Management**: Warns at 70/80/90% storage capacity with cleanup tools

### Usage

1. Navigate to any feed
2. Click "Save for Offline" button
3. Articles are cached in IndexedDB via Service Worker
4. Read cached articles even when offline
5. Changes sync automatically when back online

### Storage Limits

- **IndexedDB**: ~1GB typical, up to 60% of disk space
- **localStorage**: 5-10MB per origin
- **Cache Storage**: No hard limit, browser-managed

### Conflict Resolution

When offline changes conflict with remote updates:

- **Keep Local**: Preserve your offline edits (default)
- **Accept Remote**: Use latest version from server
- **Merge**: Keep your edits, accept content updates

## 🔍 Advanced Client-Side Search

### Overview

Instant full-text search across all cached articles powered by Web Workers. Sub-50ms query execution for 10,000+ articles.

### Features

- **Tokenization**: Smart tokenization with stop word removal
- **Inverted Index**: Efficient full-text search data structure
- **TF-IDF Scoring**: Relevance ranking using term frequency-inverse document frequency
- **Web Worker**: Background indexing without blocking UI

### Performance

- **Search Latency**: <50ms for 10k articles
- **Index Building**: Background process, non-blocking
- **Filter Updates**: <10ms for UI updates

### Search Syntax

## 🚧 Troubleshooting

### Service Worker Issues

If offline features aren't working:

1. **Check Browser Support**: Chrome 90+, Firefox 88+, Safari 14+
2. **HTTPS Required**: Service Workers require HTTPS (or localhost)
3. **Clear Cache**: Settings → Storage → Clear All Caches
4. **Check Permissions**: Allow persistent storage in browser settings

### Storage Quota Exceeded

When you hit storage limits:

1. **Check Usage**: Settings → Storage shows current usage
2. **Delete Old Articles**: Automatic cleanup for articles >90 days
3. **Clear Reading History**: Removes historical data
4. **Reduce Retention**: Lower feed retention days in settings

### Search Performance Degradation

If search becomes slow:

1. **Rebuild Index**: Settings → Search → Rebuild Index
2. **Reduce Article Count**: Delete old or unread articles
3. **Clear Search Cache**: Force worker termination and restart

## 📚 API Reference

### Search Index Manager

```typescript
import { searchIndexManager } from '@/lib/search/index-manager';

// Initialize
await searchIndexManager.initialize();

// Build index
await searchIndexManager.buildIndex();

// Search
const { results, latencyMs } = await searchIndexManager.search({
  query: 'machine learning',
  topics: ['AI'],
  limit: 50,
});

// Index single article
await searchIndexManager.indexArticle(articleId);
```

### Offline Sync Manager

```typescript
import { offlineSyncManager } from '@/lib/indexeddb/offline-sync';

// Cache articles
await offlineSyncManager.cacheArticles(feedIds, retentionDays);

// Mark as read
await offlineSyncManager.markAsRead(articleId);

// Toggle star
await offlineSyncManager.toggleStar(articleId, true);

// Resolve conflict
await offlineSyncManager.resolveConflict(
  articleId,
  'keepLocal' // or 'acceptRemote' or 'merge'
);

// Get conflicts
const conflicts = await offlineSyncManager.getConflicts();
```

### Storage Quota Monitor

```typescript
import { quotaMonitor } from '@/lib/storage/quota-monitor';

// Start monitoring
await quotaMonitor.start();

// Check quota
const status = await quotaMonitor.checkQuota();
console.log(`Usage: ${status.usageRatio * 100}%`);

// Listen for threshold events
quotaMonitor.on('threshold', (status) => {
  if (status.level === 'warning') {
    showWarningBanner(status);
  }
});
```
